Project 1: Stock Trend Prediction

Classification Problem: To buy, or not to buy, that is the question

Quick jump to original code

https://colab.research.google.com/drive/1qtUB-7rcfFENlpTvb6az1QiVkK0RYwWt?usp=sharing

Machine learning

This project is a real life problem to deal with stock market trading problem. The main purpose is to predict increasing or decreasing of the stock. We are using classification method in this project, if the stock increases, we set the class to 1, and if the stock decreased, we set the class to 0.To test the prediction result, we are going to use F1 score, and our testing is based on the 30% test data, we achieved F1 score over 52 percent.

Method I am using

Preprocessing: split the original training data set into training set and testing set, with 20% of the data is training.

Feature engineering:
First method I used is standarization, such as data normalization, however, the performance dropped. PCA also not good for this example. There is too many features and I did not find a good way to do selection. Therefore, I tried PCA to reduce dimension. I used first three dimensions, but the performance is not good. In conclusion, In the model fitting portion, I used standard preprocessing.

Algorithm:
Logestic, F1 is not good.
Decision tree, F1 increase but still cannot hit 50%
XGBoost, I achieved higher score on the leader board.
I also tried gradient boosting.
I did not try ensemble, because I think performance difference among those model are large, so I just used XGBoost result on the leaderboard.

Final Choice

I used Boost classification algorithm to solve the problem. Create a bootstrap sample based on the weights of each data. Adjust weights of data points basedon the error (misclassified increase, correct decrease) Last, combine the classifiers on the final prediction.
I tried two XGBoost model, the first one set all parameter to a default value, the second XGBoost model I increased the complexity by increased tree level. The complex model performed better.
