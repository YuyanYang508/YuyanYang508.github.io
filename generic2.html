Project 2: Diabetic Condition Forecasting

Diabetic Patient

Machine Learning

This project is about regression. We have a medical data set, the input variables correspond to measurements (physical, physiological, and blood related) for a given patient and the target variable corresponds to the level of diabetic condition in the patient. Inside the medical data set, we have three data set, x_train, x_test, and y_train. We are going to split y_train data set to two part, y_train and y_test to test our output.

Method I am using

Linear Regression
Lasso Regression

I tried multiple ways to complete this project. First, I tried the linear regression method directly. I used the scores function to test my model accuracy, the score of the Linear regression model is only 0.130441. After that, I tried to improve our work by Lasso Regression.
Lasso Regression can avoid overfitting problems. Since I am using training data to make a prediction, the output may be overfitted with train data but not the real data. Using the Lasso Regression methodology adds a penalty that can solve this problem. The penalty is the absolute magnitude of the coefficients. I tried both Lasso and Ridge, finding out that the score is similar.

Hyperparameter setting and tuning

For some model, such as LinearRegression and LassoLarsIC, I adjusted the hyperparameter manually.

For RidgeCV, which has build-in cross validation. I import lasso_CV under sklearn.linear_model. Inside the model, I set the tolerance equal to 0.1. If the updates are smaller than 0.1, the optimization code checks the dual gap for optimality and continues until it is smaller than tolerance. I set cv equals to four, which means that I have 4-fold. Meanwhile, I set random_state as its default value.

Performance Evaluation

LassoCV has the highest prediction score. Lasso Regression's performance is better than Linear Regression by using model.score(X_test, y_test) function.
